## Library Recommendation

Use `@xyflow/react` for the node editor.

Reasoning:
- The app is already React/Vite with plain CSS, and React Flow is built for React node-based editors.
- It includes dragging, zooming, panning, selection, and add/remove graph elements out of the box.
- It supports custom React nodes and handles, which is enough for the current sequential workflow model.
- Current npm metadata shows `@xyflow/react@12.11.1`, MIT license, and React peer dependency `>=17`, compatible with this repo’s React 19 setup.

References:
- React Flow docs: https://reactflow.dev/
- Handles/custom nodes: https://reactflow.dev/learn/customization/handles
- Rete.js is more processing-engine oriented and heavier for this scope: https://retejs.org/docs/
