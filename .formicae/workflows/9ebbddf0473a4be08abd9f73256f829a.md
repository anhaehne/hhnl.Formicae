## Implementation Plan

1. **Create a runtime-neutral job abstraction**
   - Add new runtime types in `src/hhnl.Formicae.Infrastructure`, e.g. `RuntimeJobSpec`, `RuntimeJobStartResult`, `RuntimeJobResult`, `RuntimeJobContextFile`, `RuntimeJobSecretFile`, `RuntimeJobSecretEnvironment`.
   - Add `IJobRuntime` with:
     - `StartJobAsync(RuntimeJobSpec spec, CancellationToken)`
     - `TryGetJobResultAsync(string externalId, CancellationToken)`
     - `ReadJobLogsAsync(string externalId, CancellationToken)`
   - Keep this below `IAgentRunner`; `WorkflowOrchestrator` should still depend only on `IAgentRunner`.

2. **Move Kubernetes behind the abstraction**
   - Refactor `KubernetesJobRunner` to implement `IJobRuntime`.
   - Keep Kubernetes-specific API/client classes internal to `Infrastructure/Kubernetes`.
   - Replace direct `KubernetesJobSpec` usage in `OpenHandsAgentRunner` and `CodexAuthSetupService` with `RuntimeJobSpec`.
   - Keep Kubernetes manifest rendering tests, but update them to use an adapter/helper if needed.
   - Verify the existing `BuildSecretEnvironment` path in `OpenHandsAgentRunner`; it is currently built but not passed into the Kubernetes spec.

3. **Add container CLI runtime options**
   - Add `ContainerRuntimeOptions` bound from `ContainerRuntime`.
   - Suggested fields:
     - `Engine`: `Docker` or `Podman`, default `Docker`
     - `Executable`: optional override, default `docker` or `podman`
     - `Image`: default worker image
     - `Network`: optional Docker/Podman network
     - `WorkspaceRoot`: host directory for mounted workspaces/context
     - `TimeoutSeconds`
     - `DeleteFinishedContainers`
     - `WorkerCallbackUrl`
     - `WorkerCallbackSecret`
   - Preserve existing `KubernetesJobOptions` for Kubernetes installs.

4. **Implement Docker/Podman runtime**
   - Add `Infrastructure/Containers/ContainerJobRuntime.cs`.
   - Execute the selected CLI via `System.Diagnostics.Process`.
   - `StartJobAsync` should run detached containers with deterministic labels, env vars, command, context-file mounts, secret-file mounts, and a generated external id.
   - `TryGetJobResultAsync` should inspect container state, return `null` while running, read logs on terminal state, detect success from exit code `0`, detect timeout using start time/options, and optionally remove finished containers.
   - `ReadJobLogsAsync` should call `docker logs` or `podman logs`.
   - Add a small `IContainerCli` wrapper so tests do not shell out.

5. **Wire runtime selection in DI**
   - Add config key `JobRuntime` or `JobRuntimeMode` with values `Container` and `Kubernetes`.
   - Default non-fake `AgentMode=OpenHands` to `Container`.
   - Register:
     - `ContainerJobRuntime` when `JobRuntime=Container` or unset.
     - `KubernetesJobRunner` when `JobRuntime=Kubernetes`.
   - Update `OpenHandsAgentRunner` constructor to depend on `IJobRuntime` and a worker image/options provider that can resolve image/callback settings without Kubernetes naming.

6. **Make Helm automatically select Kubernetes**
   - In `deploy/helm/formicae/values.yaml`, set `config.jobRuntime: "Kubernetes"`.
   - In `deploy/helm/formicae/templates/configmap.yaml`, emit `JobRuntime: Kubernetes`.
   - Keep existing `KubernetesJobs__*` values and worker callback URL defaults.
   - For raw Kubernetes manifests in `deploy/kubernetes/base/configmap.yaml`, also set `JobRuntime: Kubernetes`.

7. **Update local/default config**
   - In `src/hhnl.Formicae.Api/appsettings.json`, add `JobRuntime: "Container"` and a `ContainerRuntime` section.
   - Keep `UseFakeAdapters: true` for local fake mode.
   - Ensure manual non-fake local config can run with Docker/Podman without KIND.

8. **Add tests**
   - Update existing `OpenHandsAgentRunner` tests to assert it builds runtime-neutral specs.
   - Add unit tests for DI selection:
     - unset `JobRuntime` registers container runtime
     - `JobRuntime=Kubernetes` registers Kubernetes runtime
     - fake adapters still register `FakeAgentRunner`
   - Add container runtime tests using fake `IContainerCli`:
     - starts container with env, command, context files, and labels
     - returns `null` while running
     - returns success/failure with logs based on exit code
     - handles timeout
     - removes finished containers when configured
   - Update Helm/kustomize render tests to assert `JobRuntime: Kubernetes`.

9. **Update docs**
   - Update `README.md` runtime prerequisites: Docker or Podman for default local non-fake execution; KIND only for Kubernetes E2E.
   - Add `docs/job-runtimes.md` or extend `docs/kubernetes-deployment.md` with:
     - Docker example config
     - Podman example config
     - Kubernetes example config
     - Helm behavior: Kubernetes runtime enabled automatically
     - callback URL requirements for container runtime
     - cleanup behavior and workspace mount location

10. **Validation**
   - Run `dotnet test tests/hhnl.Formicae.Tests/hhnl.Formicae.Tests.csproj`.
   - Run Helm/template validation already covered by existing tests.
   - Optionally run Kubernetes E2E only if KIND/kubectl are available.
