## Implementation Plan

1. **Establish integration domain model**
   - Add application models for provider-agnostic integrations, e.g. `DevOpsIntegration`, `ConnectedRepository`, `IntegrationCapability`.
   - Start with GitHub-only enum/provider values, but keep names generic enough for Azure DevOps later.
   - Store:
     - provider type
     - display name
     - GitHub App client id
     - encrypted/client secret reference
     - webhook secret
     - webhook URL
     - identity-provider enabled flag
     - created/updated timestamps
   - Add EF `DbSet`s and migration in `FormicaeDbContext`.

2. **Add persistence abstraction**
   - Add `IDevOpsIntegrationStore` in the application layer.
   - Implement `EfDevOpsIntegrationStore` and in-memory fake store for local/default fake mode.
   - Cover CRUD for integrations, repository links, and identity-provider toggle state.

3. **Add GitHub App configuration service**
   - Add `DevOpsIntegrationService`.
   - Implement:
     - create GitHub integration
     - list integrations
     - get integration detail
     - update client id/client secret
     - rotate/prepopulate webhook secret
     - compute webhook URL from request base URL, e.g. `/api/webhooks/github`
   - Validate required GitHub App setup fields and return UI-friendly setup instructions:
     - callback URL
     - webhook URL
     - webhook secret
     - required repository permissions/events
   - Tests: validation, webhook secret generation, URL generation.

4. **Add integration API endpoints**
   - In `Program.cs`, add endpoints under `/api/integrations`:
     - `GET /api/integrations`
     - `POST /api/integrations/github`
     - `GET /api/integrations/{id}`
     - `PUT /api/integrations/{id}/github-app`
     - `POST /api/integrations/{id}/webhook-secret`
     - `POST /api/integrations/{id}/repositories`
     - `GET /api/integrations/{id}/repositories`
     - `PUT /api/integrations/{id}/identity-provider`
   - Return `400` for invalid GitHub settings, `404` for missing integrations, and avoid returning client secrets.

5. **Support adding repositories**
   - Add a repository connection flow that stores owner/name/url/default branch/install metadata.
   - For the first implementation, accept repository URL manually after the user installs/grants the GitHub App.
   - Add validation for GitHub repository URLs.
   - Keep the model ready for a later GitHub App installation callback or installation id.
   - Tests: valid/invalid repository URLs and duplicate repository handling.

6. **Replace static GitHub token assumptions incrementally**
   - Keep existing `GITHUB_TOKEN` fallback working.
   - Add a GitHub client factory abstraction that can create Octokit clients from:
     - existing env token
     - later GitHub App installation credentials
   - Wire existing `GitHubWorkItemProvider` and `GitHubSourceControlProvider` through that factory without changing workflow behavior yet.
   - Tests should prove existing workflow adapter tests still pass.

7. **Add ASP.NET Core Identity foundation**
   - Add `Microsoft.AspNetCore.Identity.EntityFrameworkCore`.
   - Introduce `FormicaeUser : IdentityUser`.
   - Change `FormicaeDbContext` to inherit from `IdentityDbContext<FormicaeUser>`.
   - Add Identity tables via EF migration.
   - Configure cookies and Identity in `Program.cs`.
   - Add minimal auth endpoints:
     - current user
     - logout
     - external challenge
     - external callback
   - Initially allow anonymous access to existing workflow/settings endpoints if preserving MVP behavior is necessary; then explicitly require auth only for identity-provider enabling.

8. **Implement GitHub as external identity provider**
   - Use ASP.NET Core OAuth with GitHub endpoints, or a maintained GitHub auth package if preferred by repo standards.
   - Add a startup-time GitHub auth scheme when a GitHub integration has identity-provider enabled.
   - For runtime enabling, perform a short implementation spike:
     - try registering a new scheme through `IAuthenticationSchemeProvider` plus options cache
     - if reliable, enable without restart
     - if not reliable, persist `RequiresRestart=true` and show that state in API/UI
   - Ensure the user must be authenticated before enabling GitHub as identity provider.
   - Tests: unauthenticated toggle returns `401/403`; authenticated toggle persists state.

9. **Protect app access when identity provider is enabled**
   - Once any integration has identity-provider enabled, require authenticated users for app/API access except:
     - health check
     - static assets
     - login/callback/logout endpoints
     - webhooks
   - Add an authorization policy/service that checks persisted identity-provider state.
   - Verify existing webhook handling remains unauthenticated but signature-protected.

10. **Update React UI**
   - Add a primary navigation item: `Integrations`.
   - Add GitHub integration creation form:
     - client id
     - client secret
     - generated webhook secret
     - webhook URL display/copy field
     - GitHub App setup checklist
   - Add integration detail view:
     - connected repositories list
     - add repository form
     - identity-provider enable toggle
     - restart-required warning if runtime OAuth registration is not supported
   - Update `api.ts` with typed integration endpoints.
   - Keep settings page for AI settings only.

11. **Update deployment/config docs**
   - Update README and Kubernetes/Helm docs with:
     - GitHub App creation steps
     - webhook URL and events
     - identity provider behavior
     - required public callback URL
     - restart behavior if applicable
   - Add Helm/Kubernetes values only for non-secret static auth settings if needed.
   - Do not store GitHub client secrets in ConfigMaps.

12. **Verification**
   - Run:
     - `dotnet build hhnl.Formicae.slnx`
     - `dotnet test tests/hhnl.Formicae.Tests/hhnl.Formicae.Tests.csproj`
     - `npm install` if needed in `src/hhnl.Formicae.Api/ClientApp`
     - `npm run build`
   - Manually verify:
     - create GitHub integration
     - see generated webhook URL/secret
     - add repository
     - enable identity provider while authenticated
     - login with GitHub after restart or dynamic scheme registration
     - existing workflow trigger still works.
